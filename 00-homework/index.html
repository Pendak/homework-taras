Base
git config --global user.name [імя] - Встановлює ім'я, яке буде відображатися в поле автора у виконуваних вами коммітів
git config --global user.email ''[Адреса електронної пошти]'' - Встановлює адресу електронної пошти, яка буде відображатися в інформації про виконувані вами комміти
git init [назва проекту] - Створює новий локальний репозиторій з заданим ім'ям
git status - Перераховує всі нові або змінені файли, які потребують фіксації
git add [файл] - Індексує вказаний файл для подальшого комміту
git commit -m "[повідомлення з описом]" - Фіксує проіндексовані зміни і зберігає їх в історію версій
git log - Історія коммітів для поточної гілки
git checkout [імя гілки] - Перемикається на обрану гілку і оновлює робочу директорію до її стану
git branch - Список іменованих гілок коммітів із зазначенням обраної гілки
git push [віддалений репозиторій] [гілка]- Завантажує всі зміни локальної гілки в віддалений репозиторій
git pull -Завантажує історію з віддаленого сховища та об'єднує її з локальної. pull = fetch + merge
git merge [імя гілки] - Вносить зміни зазначеної гілки в поточну гілку
git clone [url-адреса] - Викачує репозиторій разом з усією його історією змін
.gitignore файл є для того щоб Git ігнорував файли і деректорії перераховані у ньому за допомоггою wildcard синтаксису
git revert - повна протилежність git cherry-pick. Вона створює новий Комміт, який вносить зміни, протилежні вказаному коммітові, по суті скасовуючи його.
git reset [файл] - Змінює індексацію вказаного файлу, при цьому зберігаючи його вміст
git rebase [гілка] - Виконує ту саму роботу, що cherry-pick, але для ланцюжка коммітів, тим самим як би переносячи гілку на нове місце.
git cherry-pick - бере зміни, що вносяться одним коммітом, і намагається повторно застосувати їх у вигляді нового комміта в поточній гілці.
git stash - тимчасове зберігання всіх відсліджуваних файлів.
